# moco 积分消耗参考

> 所有数字基于 **200 字 prompt + 1000 字 output** 的标准估算。实际值受上下文累积、工具调用次数、模型实际计费方式影响，可能 ±30%。
> token 数为"等效 input token"（output token 已按 5x 折算成 input 量级）。
> 这只是数量级参考，不是计费承诺。

---

## 表 1：moco 各阶段调用细节

| 阶段 | 调用数 | 模型 | 单次 token 量级 | 小计 |
|---|---|---|---|---|
| **R1 初答** | 4 并发 | Sonnet 4.6 / GPT 5.4 / Gemini 3.0 Pro / DeepSeek V3.2 | ~3-6k 等效 token / 家 | **~16-24k** |
| **R2 挑战** | 4 并发 | 同 R1（prompt 含其他 3 家答案，input 偏长 1.5×） | ~5-9k / 家 | **~24-36k** |
| **R3 反驳** | 平均 2.5 | 被挑战方（同 R1 模型） | ~4-7k | **~10-18k** |
| **R4 裁判** | 平均 2.5 | `claude-opus-4.7-1m`（独立第 5 家，不在参战） | ~6-10k | **~15-25k** |
| **WorkBuddy 主智能体编排** | 5-8 次 | **当前会话模型**（如 `claude-opus-4.7-1m` / `claude-sonnet-4.6-1m` / `gpt-5.4` 等，由用户在 WorkBuddy 顶部选择器决定） | ~3-8k / 次 | **~20-65k** |
| **R1 视觉 token**（仅 r1_only） | 4 | 视觉编码 | ~1-2k / 家 | **~4-8k** |

**注**：
1. 主智能体编排成本"自动跟随会话当前模型"——不在 moco 代码里写死。在 WorkBuddy 顶部切到 Sonnet 4.6 1M 即可减半，**无需改任何 moco 代码**。
2. 表中"单次 token 量级"是基于 200 字 prompt + 1000 字 output 估算，实际可能 ±30%。
3. 不同厂商的模型对相同 token 量的实际计费策略不同（积分换算系数有差异）。

## 表 2：消耗对照（含基准）

| 模式 | 等效 token 总量（量级）| 相对单次质询 | 备注 |

2. 月度可跑次数取决于你的 WorkBuddy 配额——具体换算请参考 WorkBuddy 后台的"积分/token"对照。

## 优化建议

1. **moco 是大投入**（≈ 20× 单次质询量级），慎用但不奢侈
2. **真正烧 token 的不是 moco，是长对话**——一次 90+ 轮主对话 ≈ 1 次 moco
3. **r1_only vs 纯文字差距很小**（< 10%），有图就直接 r1_only，不用犹豫
4. **日常切到 Sonnet 主智能体**：moco 编排部分自动减半，且不影响参战阵容质量
